FlashAttention V1–V3 优化详解

FlashAttention 通过 Tiling + Online Softmax 消除了标准 Attention 中 O(N²) 的 HBM 读写瓶颈， 从 V1 的 IO-aware 算法到 V3 的 Hopper 异步流水线，每一代都在工程化利用 GPU 硬件上跨越了一步。 本文档覆盖核心算法原理、分块策略（含配图）、以及各版本的关键优化。

V1: Tiling + Online Softmax V2: 循环重排 + Warp 并行 V3: Hopper Warp Specialization IO 复杂度: O(N²d² / M)

1. 为什么需要 FlashAttention

标准 Self-Attention 计算 O = softmax(QKT) · V，需要把 N×N 的 Attention Matrix 先完整写入 HBM， 再读回做 softmax 和与 V 相乘。当序列长度 N 增大时，HBM 读写量以 O(N²) 增长，成为性能瓶颈。

标准 Attention 的 HBM 访问

写入 S = QKT (N×N) → 读回 S → 写入 P = softmax(S) → 读回 P → 写入 O = PV

总 HBM 访问: O(N² + Nd) — 被 N² 主导

FlashAttention 的 HBM 访问

Q, K, V 分块加载到 SRAM → 在 SRAM 中完成 QKT → softmax → ×V → 只写回最终 O

总 HBM 访问: O(N²d² / M) — M 为 SRAM 大小

图：标准 Attention 中 S、P 两个 N×N 矩阵在 HBM 中的写回与读回构成 IO 瓶颈

2. Tiling 分块策略

FlashAttention 沿 序列长度维度 对 Q、K、V 进行分块。设序列长度为 N，head dim 为 d，选取 Q 方向 block size 为 Br，K/V 方向为 Bc：

Q ∈ ℝ^{N×d} → 分成 ⌈N/Br⌉ 块: Q₁(Br×d), Q₂(Br×d), ... K ∈ ℝ^{N×d} → 分成 ⌈N/Bc⌉ 块: K₁(Bc×d), K₂(Bc×d), ... V ∈ ℝ^{N×d} → 分成 ⌈N/Bc⌉ 块: V₁(Bc×d), V₂(Bc×d), ...

图：FlashAttention 的 Tiling 分块全景 — Q 沿行分块 (Br)，K/V 沿列分块 (Bc)，高亮区域为当前 SRAM 中处理的块

3. 分块对比：Attention Tiling vs GEMM Tiling

GEMM 的分块可以沿 M、N、K 三个维度自由切分，partial sum 直接累加。 FlashAttention 在 K/V 序列维分块时，由于 softmax 是行级全局归一化，无法简单累加——这是核心区别。

图：GEMM Tiling（左）vs FlashAttention Tiling（右）— 关键区别在于 softmax 导致 K 维分块时旧结果需要 rescale

GEMM TilingFlashAttention Tiling
分块维度M × N × K 三维自由分Q 方向 (Br) × K/V 方向 (Bc)
累加方式partial sum 直接相加需 rescale：旧 O 乘 e(m_old − m_new)
SRAM 存储A tile + B tile + C partial sumQ_i + K_j + V_j + O_i + m_i + l_i
核心难点无online softmax 的增量更新
Block 间依赖无（可乱序累加）沿 K/V 方向有序（需携带 m, l 状态）

4. Online Softmax 算法

Online Softmax 是 FlashAttention 分块可行的数学基础。它允许在只看到部分数据的情况下增量计算 softmax 归一化， 每当新数据到来时只需更新 max 和 sum，并对旧的结果做一次 rescale。

图：Online Softmax 的增量计算流程 — 每处理一个新 K/V block，更新 m, l 并 rescale 旧输出

// Online Softmax 伪代码 — 处理第 j 个 K/V block S_ij = Q_i @ K_j.T // (Br × Bc) m_new = max(m_old, rowmax(S_ij)) // 更新行最大值 alpha = exp(m_old - m_new) // 旧结果的修正因子 l_new = alpha * l_old + rowsum(exp(S_ij - m_new)) // 更新归一化因子 O_i = alpha * O_i + exp(S_ij - m_new) @ V_j // rescale + 累加 m_old = m_new l_old = l_new // 所有 block 遍历完后: O_i = O_i / l_new // 最终归一化

5. FlashAttention V1

V1 (Dao et al., 2022) 首次将 Tiling + Online Softmax 引入 Attention 计算。 核心思想：将 Q、K、V 分块加载到 SRAM，在片上完成完整的 softmax + matmul， 避免 N×N 中间矩阵写回 HBM。

核心技术
  • Tiling：Q 分 ⌈N/Br⌉ 块，K/V 各分 ⌈N/Bc⌉ 块
  • Online Softmax：增量维护 (m, l) 对，支持分块归一化
  • Recomputation：反向传播不存 attention matrix，从 Q/K/V/O 重算，用计算换显存
V1
V1 循环结构
// V1: 外层遍历 K/V blocks，内层遍历 Q blocks for j in range(⌈N/Bc⌉): // 外层: K/V block load K_j, V_j to SRAM for i in range(⌈N/Br⌉): // 内层: Q block load Q_i, O_i, m_i, l_i to SRAM compute S_ij, update O_i // online softmax write O_i, m_i, l_i back to HBM ← 每轮都写回!

问题：O_i 每个内层迭代都要写回 HBM

IO 复杂度：从标准 Attention 的 Θ(Nd + N²) 降低到 Θ(N²d² / M)，其中 M 为 SRAM 大小。 当 M = Θ(Nd) 时，IO 为 Θ(Nd)，接近线性。
GPU 利用率

A100 上 ~25–40% 峰值 FLOPS

内存节省

不存储 N×N attention matrix，显存从 O(N²) 降至 O(N)

训练加速

BERT-large 约 15%，GPT-2 约 3× 加速

6. FlashAttention V2

V2 (Dao, 2023) 在算法不变的前提下，通过三类工程优化将 A100 上的 GPU 利用率从 ~30% 提升到 ~70%。

优化 ①：减少非 matmul FLOPs

V1 每处理一个 K/V block 就做一次 rescale（乘 diag(em_old−m_new)），产生大量非 matmul 的 FLOPs。 V2 推迟 rescale 到所有 block 结束后一次性归一化，减少中间的 exp/div 操作。

// V2: 延迟归一化 // 循环中不做 O = O / l 只维护未归一化的 O_acc for each K/V block: O_acc = exp(m_old - m_new) * O_acc + exp(S - m_new) @ V // 循环结束后: O = diag(1/l_final) * O_acc // 一次性归一化
优化 ②：交换循环顺序
优化 ③：Warp 级并行
V1: Split-K warp 策略

不同 warp 处理 K/V 的不同 split → 各 warp 计算 partial O → 需通过 shared memory 做 reduce → 同步 + shared memory 读写开销大

V2: Split-Q warp 策略

不同 warp 处理 Q 的不同行 → 各 warp 独立拥有完整的 O_i → 无需 warp 间通信，无 reduce 开销

优化 ④：因果遮罩跳过

在 causal attention 中，当 Q block 的 token 位置全部早于 K block 的 token 位置时， 该 S block 全部被 mask 为 −∞，可以跳过整个 block 的计算。理论上 causal attention 约快 1.7–1.8×。

峰值利用率：A100 上约 70% 理论峰值 FLOPS（V1 仅 25–40%）， forward 约 2×加速，backward 约 1.7×，端到端训练约 2× 加速。

7. FlashAttention V3

V3 (Shah et al., 2024) 针对 NVIDIA Hopper 架构（H100）的硬件特性做深度适配。 核心理念：利用 Hopper 新指令实现异步流水线 + 计算/搬运 overlap。

Warp Specialization — 生产者/消费者流水线
  • Producer warp：使用 TMA (Tensor Memory Accelerator) 异步地将 K/V blocks 从 HBM 搬到 Shared Memory
  • Consumer warp：使用 WGMMA 指令在 Tensor Core 上做 QKᵀ 和 PV 的矩阵乘
  • 两者通过 barrier 同步，形成流水线 —— 搬运和计算完全 overlap
Softmax 与 GEMM 的 Pingpong Overlap

Softmax 中的 exp / sum / div 使用 CUDA Core（math pipeline）， 而 GEMM 使用 Tensor Core（矩阵乘 pipeline）。两者硬件单元不同，可以并行：

WGMMA 指令

Hopper 的 Warp Group Matrix Multiply-Accumulate 指令可直接从 Shared Memory 发起矩阵乘法到 Tensor Core， 绕过寄存器文件的中转，提高吞吐量。V3 的所有 GEMM（QKᵀ 和 PV）都使用 WGMMA。

FP8 低精度支持
  • Block Quantization：对每个 tile 使用独立的 scale factor，而非全局量化，减少精度损失
  • Incoherent Processing：对 Q/K 乘以随机正交矩阵，使异常值分散到各维度，提高量化后的数值稳定性

FP8 模式下 H100 可达约 1.2 PFLOPS 的 attention 吞吐。

H100 性能：FP16 达到 ~740 TFLOPS（~75% 峰值利用率），相比未优化的 Hopper FlashAttention-2 提升 1.5–2.0×。
