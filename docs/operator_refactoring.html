算子层重构方案

当前算子层的核心问题是 kernel_ops.cu 作为一个 1547 行的单体文件承载了 14 个 CUDA kernel、全部 launch 函数和公共验证逻辑， 同时 layers/ 目录混合了无状态 kernel 文件与有状态的 Module wrapper。本文档给出一套拆分方案，把无状态 kernel 按功能域拆进 kernels/ 子目录，让 layers/ 专注于有状态算子。

kernel_ops.cu: 1547 行单体文件 layers/ 混合: kernel + Module 目标: kernels/ 无状态 + layers/ 有状态 不改变公共 API 签名

现状与问题

当前 src/layers/ 目录同时承担两种角色: (1) 无状态 CUDA kernel 集合，(2) 有状态的 Module wrapper。这导致几个具体问题:

单体膨胀 kernel_ops.cu 已达 1547 行，包含 14 个 __global__ kernel + 12 个 launch wrapper + 12 个公共 API 函数 + 通用 validation 辅助。任何新增 kernel 都只能往末尾追加。
职责混淆 src/layers/ 放了 kernel_ops.cu、attention.cu (纯 CUDA kernel)，也放了 activation.cpp、linear.cpp (有状态 Module)。 二者的抽象层次完全不同，混在一起增加理解成本。
编译耦合 修改 rms_norm kernel 就要重新编译整个 kernel_ops.cu (1547 行)，nvcc 编译时间随文件大小非线性增长。 拆分后每个 .cu 独立编译，增量构建更快。
当前文件清单
include/layers/ ├── activation.h # Module: SiluAndMul wrapper ├── attention.h # Module: Attention (有状态, 持有 kv_cache 引用) ├── embed_head.h # Module: Embedding + LM Head ├── kernel_ops.h # 🔴 所有无状态 kernel 的统一头文件 (12 个函数声明) ├── layernorm.h # Module: RMSNorm wrapper ├── linear.h # Module: Linear 家族 ├── rotary_embedding.h # Module: RotaryEmbedding wrapper └── sampler.h # Module: Sampler src/layers/ ├── activation.cpp # Module 实现 ├── attention.cpp # Module 实现 (调用 attention.cu kernel + kernel_ops) ├── attention.cu # 🔴 3 个 __global__ attention kernel (731 行) ├── embed_head.cpp # Module 实现 ├── kernel_ops.cpp # phase1_kernel_specs() 元数据 ├── kernel_ops.cu # 🔴 14 个 __global__ kernel (1547 行, 核心痛点) ├── layernorm.cpp # Module 实现 ├── linear.cpp # Module 实现 ├── rotary_embedding.cpp # Module 实现 └── sampler.cpp # Module 实现

目标目录结构

核心思路: 新建 kernels/ 目录存放所有无状态 CUDA kernel，layers/ 目录只保留有状态的 Module wrapper 和它们的 .cpp 实现。

重构前
include/layers/kernel_ops.h # 统一头文件 src/layers/kernel_ops.cu # 1547 行全部 kernel src/layers/attention.cu # 731 行 attention kernel src/layers/kernel_ops.cpp # 元数据
全部挤在 layers/
重构后
include/kernels/ ├── kernel_types.h # 公共类型 + validation ├── rms_norm.h ├── rotary.h ├── activation.h ├── bias_add.h ├── gather.h ├── embedding.h ├── sampling.h ├── kvcache.h ├── attention.h └── split_qkv.h src/kernels/ ├── kernel_types.cu # 公共 validation + specs ├── rms_norm.cu ├── rotary.cu ├── activation.cu ├── bias_add.cu ├── gather.cu ├── embedding.cu ├── sampling.cu ├── kvcache.cu ├── attention.cu └── split_qkv.cu
按功能域独立文件
layers/ 保留的文件
include/layers/ ├── activation.h # Module: SiluAndMul ├── attention.h # Module: Attention ├── embed_head.h # Module: Embedding + LM Head ├── kernel_ops.h # ⚠️ 保留为 umbrella header, #include 所有 kernels/*.h ├── layernorm.h # Module: RMSNorm ├── linear.h # Module: Linear 家族 ├── rotary_embedding.h # Module: RotaryEmbedding └── sampler.h # Module: Sampler src/layers/ ├── activation.cpp ├── attention.cpp # 调用 kernels/attention.h 而非直接依赖 .cu ├── embed_head.cpp ├── layernorm.cpp ├── linear.cpp ├── rotary_embedding.cpp └── sampler.cpp

kernel_ops.h 保留为 umbrella header，内部改为 #include 所有 kernels/*.h 头文件。 这样已有的 #include "layers/kernel_ops.h" 代码无需改动。

头文件设计

每个 kernels/*.h 只暴露该功能域的公共 API 函数，不暴露 __global__ kernel 和 launch wrapper。 kernel_types.h 提供公共类型、validation 函数声明和 dtype dispatch 宏。

kernels/kernel_types.h 示例
#pragma once #include "utils/tensor.h" #include <cstdint> #include <vector> namespace nano_vllm { // ---- Kernel metadata ---- struct KernelSpec { const char* name; const char* purpose; }; const std::vector<KernelSpec>& phase1_kernel_specs(); // ---- Validation helpers (定义在 kernel_types.cu) ---- void ensure_cuda_tensor(const char* name, const Tensor& tensor); void ensure_same_device(const char* lhs_name, const Tensor& lhs, const char* rhs_name, const Tensor& rhs); void ensure_same_shape(const char* lhs_name, const Tensor& lhs, const char* rhs_name, const Tensor& rhs); void ensure_same_dtype(const char* lhs_name, const Tensor& lhs, const char* rhs_name, const Tensor& rhs); void validate_supported_dtype(const char* name, ScalarType dtype); // ---- Dtype dispatch macros ---- // DISPATCH_FLOAT_TYPES(dtype, "name", [&] { ... }) // DISPATCH_FLOAT_TYPES_NO_F32(dtype, "name", [&] { ... }) ... } // namespace nano_vllm
kernels/rms_norm.h 示例
#pragma once #include "utils/tensor.h" namespace nano_vllm { void rms_norm(const Tensor& input, const Tensor& weight, float eps, Tensor& output, cudaStream_t stream = nullptr); void add_rms_norm(const Tensor& input, const Tensor& residual, const Tensor& weight, float eps, Tensor& output, Tensor& residual_out, cudaStream_t stream = nullptr); } // namespace nano_vllm

注意: __global__ kernel 和 launch_ wrapper 只出现在 .cu 文件内部，不进入头文件。 头文件只包含最终的公共 API，签名与现有 kernel_ops.h 完全一致。

layers/kernel_ops.h — 保留为 umbrella header
#pragma once // Umbrella header — 向后兼容已有代码 // 已有的 #include "layers/kernel_ops.h" 无需修改 #include "kernels/kernel_types.h" #include "kernels/rms_norm.h" #include "kernels/rotary.h" #include "kernels/activation.h" #include "kernels/bias_add.h" #include "kernels/gather.h" #include "kernels/embedding.h" #include "kernels/sampling.h" #include "kernels/kvcache.h" #include "kernels/split_qkv.h"

kernel_ops.h 退化为纯转发头文件，所有上游 #include "layers/kernel_ops.h" 的代码零改动。 新写的代码可以直接 #include 具体的 kernels/*.h，减少编译依赖。

Layers 层保留项

重构后 layers/ 只包含有状态的 Module wrapper。每个 Module 通过 #include 对应的 kernels/*.h 调用底层 kernel，不再直接持有 CUDA 代码。

Module 头文件 实现 依赖的 kernels/
RMSNorm layers/layernorm.h layers/layernorm.cpp kernels/rms_norm.h
RotaryEmbedding layers/rotary_embedding.h layers/rotary_embedding.cpp kernels/rotary.h
SiluAndMul layers/activation.h layers/activation.cpp kernels/activation.h
Attention layers/attention.h layers/attention.cpp kernels/attention.h, kernels/kvcache.h
Linear 家族 layers/linear.h layers/linear.cpp (cuBLAS wrapper, 无直接 kernel 依赖)
Embedding + LM Head layers/embed_head.h layers/embed_head.cpp kernels/embedding.h, kernels/gather.h
Sampler layers/sampler.h layers/sampler.cpp kernels/sampling.h
Module 的职责
  • 持有权重 Tensor 引用 (weight, bias)
  • 持有算子参数 (eps, num_heads, head_dim...)
  • forward() 中组合 kernel 调用 + cuBLAS
  • 管理 KV cache 生命周期 (Attention)
kernels/ 的职责
  • 无状态: 不持有任何 weight 或缓存
  • 输入输出全部通过函数参数传递
  • 只依赖 utils/tensor.h 和自身的 kernel_types.h
  • 每个 .cu 文件可独立编译

执行步骤

按以下顺序操作，每一步结束后确保 ctest 全部通过，再进入下一步。

Step 1: 创建 kernels/ 目录 + 公共基础
预计改动: 4 个新文件
  • 创建 include/kernels/ 和 src/kernels/ 目录
  • 从 kernel_ops.cu 行 1–210 提取 validation 函数和 dispatch 宏到 kernel_types.h / kernel_types.cu
  • 从 kernel_ops.cpp 移动 phase1_kernel_specs() 到 kernel_types.cu
  • CMakeLists.txt 添加 src/kernels/kernel_types.cu
  • 验证: cmake --build build && ctest --test-dir build
mkdir -p include/kernels src/kernels # 提取公共验证逻辑 # 新建 include/kernels/kernel_types.h # 新建 src/kernels/kernel_types.cu
Step 2: 拆分 rms_norm kernel
从 kernel_ops.cu 提取 4 个 kernel + 4 个 launch + 2 个 API
  • 从 kernel_ops.cu 提取: rms_norm_kernel, rms_norm_f32x4_kernel, add_rms_norm_kernel, add_rms_norm_f32x4_kernel
  • 以及对应的 launch_rms_norm, launch_rms_norm_f32, launch_add_rms_norm, launch_add_rms_norm_f32
  • 以及公共 API: rms_norm(), add_rms_norm()
  • 目标: src/kernels/rms_norm.cu + include/kernels/rms_norm.h
  • 从 kernel_ops.cu 删除对应代码
  • 验证: cmake --build build && ctest --test-dir build
Step 3: 拆分 rotary kernel
从 kernel_ops.cu 提取 2 个 kernel + 2 个 launch + 1 个 API
  • 提取: apply_rotary_kernel, apply_rotary_f32x2_kernel
  • 以及 launch_rotary_one, launch_rotary_one_f32, rotary_embedding()
  • 目标: src/kernels/rotary.cu + include/kernels/rotary.h

Step 5: 迁移 attention kernel

  • 提取公共 API 声明到 include/kernels/attention.h

Step 6: 更新 kernel_ops.h 为 umbrella header

  • 删除已清空的 src/layers/kernel_ops.cu 和 src/layers/kernel_ops.cpp
  • 删除 src/layers/attention.cu (已移动到 src/kernels/)
  • CMakeLists.txt 删除旧文件, 确认新文件全部在 CUDA_SOURCES 列表
  • 最终验证: cmake --build build && ctest --test-dir build —— 10/10 通过

CMake 适配

CMakeLists.txt 需要把旧的 .cu 源文件替换为新的 kernels/ 下的源文件列表。

CMake 变更对比
# ---- 删除 ---- src/layers/kernel_ops.cu src/layers/kernel_ops.cpp src/layers/attention.cu # ---- 新增 ---- src/kernels/kernel_types.cu src/kernels/rms_norm.cu src/kernels/rotary.cu src/kernels/activation.cu src/kernels/bias_add.cu src/kernels/gather.cu src/kernels/embedding.cu src/kernels/sampling.cu src/kernels/kvcache.cu src/kernels/attention.cu src/kernels/split_qkv.cu

include 路径已覆盖 include/ 根目录，因此 #include "kernels/rms_norm.h" 可直接解析，无需额外 include_directories。

建议的 CUDA_SOURCES 变量
set(KERNEL_SOURCES src/kernels/kernel_types.cu src/kernels/rms_norm.cu src/kernels/rotary.cu src/kernels/activation.cu src/kernels/bias_add.cu src/kernels/gather.cu src/kernels/embedding.cu src/kernels/sampling.cu src/kernels/kvcache.cu src/kernels/attention.cu src/kernels/split_qkv.cu ) set(LAYER_SOURCES src/layers/activation.cpp src/layers/attention.cpp src/layers/embed_head.cpp src/layers/layernorm.cpp src/layers/linear.cpp src/layers/rotary_embedding.cpp src/layers/sampler.cpp )

将 CUDA kernel 源文件和 C++ Module 源文件分开管理，便于后续为 kernel 设置不同的编译选项 (如 -maxrregcount、-use_fast_math 等)。

测试策略

重构过程中不改变任何公共 API 签名，因此现有的 10 个测试用例无需修改。 以下策略确保重构不引入回归:

每步验证
  • 每拆出一个 kernel 文件后立即 cmake --build + ctest
  • 10/10 测试全部通过才进入下一步
  • 如果 link error，检查 CMakeLists.txt 是否遗漏新 .cu 文件
  • 如果 undefined reference，检查 namespace 和函数签名是否一致
端到端验证
  • 最终完成后运行 bench_e2e 确认 decode 吞吐 ≥ 85 tok/s
  • 运行 example.cpp 确认聊天输出正常
  • 对比 Python 输出确认 token-for-token 一致
  • 增量编译时间应明显减少 (修改单个 kernel 只重编那个 .cu)
相关测试文件
测试文件 覆盖范围 与 kernels/ 的关系
tests/test_tensor.cpp Tensor 运行时 + 基础 kernel 间接依赖 kernels/ (通过 layers/kernel_ops.h umbrella)
tests/test_layers.cpp Context + Attention + Linear + Embedding + Sampler 通过 Module wrapper 间接调用 kernels/
tests/test_models.cpp Qwen3 模型端到端推理 全链路依赖
bench/bench_e2e.cpp 5-prompt decode 吞吐基准 全链路依赖

风险与约束

低风险
  • 公共 API 签名不变，所有上游代码零改动
  • kernel_ops.h umbrella header 保持向后兼容
  • 每步可独立验证，回退只需 git checkout
需要注意
  • static 函数 / 匿名 namespace 内的辅助函数可能在多个 kernel 间共用，拆分时需确认依赖关系
  • kernel_ops.cu 内的 #include 顺序和 CUDA header 依赖需要在每个新 .cu 中正确复制
  • 编译顺序: kernel_types.cu 必须先编译 (提供 validation 符号)
  • attention.cu 中如果有对 kernel_ops.cu 内辅助函数的内部调用，需改为 #include "kernels/kernel_types.h"
不在本次重构范围内的事项
  • 不引入 bf16 dtype 支持 (这是独立的优化项)
  • 不替换 Attention 实现为 FlashAttention (需要确认第三方库集成)
  • 不改变 Module wrapper 的接口或行为
  • 不修改 tests/ 中的任何测试用例
  • 不引入 kernel fusion 或新的优化策略
本方案的核心原则: 纯文件组织重构，不改变运行时行为，每步可验证可回退。完成后 kernel_ops.cu 的 1547 行将分散到 11 个独立 .cu 文件中， 平均每个文件约 140 行，增量编译和代码导航体验大幅改善。
